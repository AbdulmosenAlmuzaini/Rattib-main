# رتّب

منصة عربية لإدارة مكاتب الخدمات العامة والمعقّبين: العملاء، المعاملات، المهام، المستندات، الدفعات، المصروفات وسجل التدقيق.

المشروع مستقل عن Google AI Studio ولا يحتاج Gemini لتشغيل وظائفه الأساسية.

## التشغيل المحلي

المتطلبات: Node.js 20 أو أحدث وPostgreSQL 17. يتضمن المشروع إعداد Docker محلياً.

1. انسخ `.env.example` إلى `.env.local` واضبط كلمة مرور تطوير طويلة.
2. ثبّت الحزم: `npm install`.
3. شغّل PostgreSQL وMinIO: `docker compose up -d postgres minio` أو شغّل الحاويتين `rattib-postgres` و`rattib-minio` الموجودتين محلياً.
4. طبّق الترحيلات: `npm run db:deploy`.
5. شغّل المنصة: `npm run dev`.
6. سجّل الدخول بالبريد `owner@rattib.com` وكلمة المرور المحددة في `SEED_ADMIN_PASSWORD`.

## فحوص الجودة

- `npm run lint`: فحص TypeScript.
- `npm run build`: بناء نسخة الإنتاج.
- `npm run test:unit`: اختبار التشفير وخريطة الصلاحيات.
- `npm run test:integration`: إنشاء مخطط PostgreSQL معزول ثم اختبار الدخول، عزل المكاتب، الصلاحيات، التحقق من الطلبات والأداء.
- `npm test`: تشغيل جميع الاختبارات الآلية.

لا تستخدم اختبارات التكامل بيانات التشغيل؛ فهي تعيد إنشاء المخطط المعزول `rattib_test` فقط.

## الصحة التشغيلية

- `GET /healthz`: يتحقق من أن عملية الخادم تعمل.
- `GET /readyz`: يتحقق من اتصال PostgreSQL ويعيد `503` إذا لم تكن قاعدة البيانات جاهزة.

## النسخ الاحتياطي والاستعادة

- `npm run backup`: ينشئ نسخة مؤرخة من مخطط PostgreSQL الإنتاجي ومستندات S3/MinIO مع بصمات SHA-256 داخل مجلد `backups`.
- `npm run backup:verify -- -BackupDirectory <المجلد>`: يفحص البصمات ويستعيد قاعدة البيانات فعلياً داخل قاعدة مؤقتة ثم يحذف قاعدة التحقق.
- الاستعادة الفعلية تتطلب أمراً صريحاً يحمي من الاستبدال العرضي:
  `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/restore.ps1 -BackupDirectory <المجلد> -Database <القاعدة> -ConfirmDatabaseOverwrite`

ينبغي في الإنتاج جدولة النسخ يومياً، تشفيرها، نقل نسخة إلى موقع منفصل، وتجربة الاستعادة دورياً.

## ضوابط مهمة

- قاعدة البيانات المعتمدة هي PostgreSQL، وملف SQLite القديم محفوظ مؤقتاً للرجوع أثناء التطوير فقط.
- لا تستخدم كلمة مرور التطوير في الإنتاج.
- مسار إعادة البيانات التجريبية معطل عندما تكون `NODE_ENV=production`.
- جميع واجهات البيانات تتطلب جلسة دخول وتستخرج المكتب من الجلسة، لا من المتصفح.
- المستندات تُخزن في مخزن S3 خاص، ولا توجد روابط ملفات عامة. الرفع محدود بـ5MB ويقبل PDF وPNG وJPEG بعد فحص بصمة المحتوى.
- مولد التنبيهات يعمل عند بدء الخادم ثم كل ساعة، ويمنع تكرار تنبيهات المستندات والمهام والمتابعات.

## حالة المشروع

الواجهة والوحدات الأساسية وطبقات المصادقة والعزل والصلاحيات والتخزين والاختبارات والنسخ الاحتياطي ومسار النشر الإنتاجي موجودة ومختبرة محلياً. يتطلب الإطلاق العام فقط إدخال نطاق الخادم وأسرار الإنتاج الخاصة بالجهة ثم اتباع بوابة القبول.

تعليمات النشر والتحديث والمراقبة والاستعادة موثقة في [دليل تشغيل الإنتاج](./PRODUCTION_RUNBOOK.md).
